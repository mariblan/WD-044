Chapter Projects in Github

Instructions

  1. Each group should create a branch with the name feature/group-name who worked on the chapter projects:
    • Chapter project HTML
    • Chapter project CSS
  2. Create a folder with the groups name in the Primer>Week-1>chapter-projects.
  3. The other team members have to pull feature/group-name in the computer.
  4. From this branch each one is going to create a new individual branch name feature/nameOfSections (use your names and i will send my cat in the night to eat your socks!).
    • The next sections should be the ones your team should develope int each branch.
      • index/main
      • index/navfooter
      • contact
      • styles/navfooter
      • styles/main
  5. Copy paste in the branch the part of the code of the project that is refered by after the slash of feature/nameOfSections.
  6. When all is done, push the feature/nameOfSections to the remote.
  7. Merge branches one by one, from top to bottom in the order of step 4.
    • The person working on the feature/index/main creates a pull request for it to merge with the feature/group-name, and merge it when it has three approvals.
    • Each time the branch of a teammate is added, the person who has to add the next branch has to:
      • Pull the feature/group-name in their branch.
      • Solve the merge conflicts.
      • Commit and push the code to update the branch on the remote
      • Create a pull request for that branch and merge it once it has three approvals.
  8. When all branches have been merged with feature/group-name, the group should create a pull request to merge the group branch with dev.
